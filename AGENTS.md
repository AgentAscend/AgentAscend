# AgentAscend - Hermes Agent Configuration

## Overview
This project is a structured AI agent system called AgentAscend.

The system uses:
- Hermes Agent for reasoning and execution
- Obsidian as a knowledge base interface
- A structured folder system for memory and learning
- Token-gated monetization using blockchain payments

You are operating inside this system and must follow its rules strictly.

---

## Directory Structure

The project uses three main folders:

/raw/     → Unprocessed notes, scraped data, and incoming information  
/wiki/    → Structured knowledge pages (clean, organized, linked)  
/system/  → Rules, schema, and system-level definitions  

DO NOT mix these folders.

---

## Knowledge Handling Rules

### RAW (/raw)
- Store unprocessed information here
- Includes:
  - ideas
  - scraped content
  - external documents
- Do NOT structure or clean content here

---

### WIKI (/wiki)
- Store structured, clean knowledge here
- Every page MUST follow schema.md
- Use markdown format
- Use [[wikilinks]] to connect related pages
- One concept per page

---

### SYSTEM (/system)
- Contains system-level files such as:
  - purpose.md
  - schema.md
- These define how the system behaves
- Do NOT write normal knowledge pages here

---

## Page Creation Rules

When creating a wiki page:

1. Use this structure:

## Summary
Short explanation

## Components
List of important parts

## Relationships
Use [[wikilinks]]

## Notes
Additional important information

---

2. Naming Rules:
- Use clear names (e.g. "Payment System")
- Do NOT use versioning (no v1, v2)
- Avoid duplicate pages

---

3. Linking Rules:
- Always link related concepts
- Build a connected knowledge graph
- Avoid isolated pages

---

## Workflow Rules

When new information is received:

1. Save raw version in /raw
2. Analyze and extract key concepts
3. Create or update structured wiki page in /wiki
4. Link it to existing pages

---

## Behavior Rules

- Do NOT overwrite existing wiki pages unless explicitly instructed
- Do NOT create duplicate concepts
- Do NOT store raw data in /wiki
- Do NOT store structured data in /raw
- Always prioritize clarity and structure
