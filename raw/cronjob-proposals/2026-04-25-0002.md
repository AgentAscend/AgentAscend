# Cronjob Proposal: MEMORY.md Maintenance

Date: 2026-04-25-0002
Proposed by: Reuben / Hermes
Source finding: User requested a recurring MEMORY.md maintenance cronjob after defining AgentAscend split-brain autonomy rules.

## Reason
MEMORY.md can become stale, too long, or polluted with temporary details as AgentAscend evolves. A recurring report-only maintenance review keeps project memory concise, accurate, and safe without automatically changing durable memory.

## Trigger Condition
Daily evening review, and optionally after major changes to reports, git state, llm-wiki/raw/wiki/system, payment architecture, or roadmap direction.

## Schedule
Daily evening after the existing daily evening audit cycle.

Recommended cron expression:
`30 19 * * *`

## Model Tier
Reasoning

## Risk Level
Low to Medium

Low because it is report-only and proposes patches. Medium because MEMORY.md can influence future agent behavior if edited incorrectly.

## Inputs
- `/home/agentascend/projects/AgentAscend/MEMORY.md`
- `/home/agentascend/projects/AgentAscend/raw/`
- `/home/agentascend/projects/AgentAscend/wiki/`
- `/home/agentascend/projects/AgentAscend/system/`
- Git status and recent diffs in `/home/agentascend/projects/AgentAscend`
- Recent payment-system, roadmap, and architecture reports if present

## Outputs
- `/home/agentascend/projects/AgentAscend/raw/memory-maintenance/YYYY-MM-DD.md`

## Exact Cronjob Prompt
You are Hermes, the autonomous project operator for AgentAscend. Run the MEMORY.md Maintenance cronjob as report-only.

Project root: `/home/agentascend/projects/AgentAscend`.
Map `llm-wiki/raw/...` to the repo-level `raw/...` directory unless a separate llm-wiki directory exists.

Review AgentAscend reports, git changes, raw/wiki/system updates, payment system changes, and roadmap changes.

Check whether `MEMORY.md` is:
1. Accurate
2. Current
3. Too long
4. Missing important architecture decisions
5. Carrying outdated assumptions
6. Missing current priorities
7. Accidentally storing anything sensitive

If updates are needed, propose a patch to `MEMORY.md` in the report. Do not apply the patch automatically.

Rules:
- Read before writing.
- Do not add secrets.
- Do not add raw logs.
- Do not add temporary debugging noise.
- Keep MEMORY.md concise and high-signal.
- Do not modify `MEMORY.md` automatically.
- Do not modify code.
- Do not commit or push.
- Redact any sensitive values found.

Save a clear markdown report to:
`raw/memory-maintenance/YYYY-MM-DD.md`

Report format:
1. Summary
2. Files/reports reviewed
3. MEMORY.md accuracy check
4. Stale/outdated assumptions
5. Missing high-signal facts
6. Overlong/noisy items
7. Sensitive-data concerns
8. Proposed patch, if needed
9. Recommended next action for Reuben

## Allowed Actions
- Read files
- Inspect git status/diffs
- Create one markdown report under `raw/memory-maintenance/`
- Propose a patch to MEMORY.md inside the report

## Forbidden Actions
- Do not edit MEMORY.md automatically
- Do not expose secrets
- Do not write raw logs into MEMORY.md
- Do not store temporary debugging noise
- Do not modify payment, wallet, access-control, database, or deployment code
- Do not commit, push, deploy, post externally, send emails, or send appeals

## Escalation Condition
Escalate to Hermes/Premium Strategic if the review finds:
- Memory instructions that could weaken payment verification
- Outdated payment/access/security assumptions
- Secrets or credentials in MEMORY.md
- Conflicting autonomy rules
- Public launch, tokenomics, or user-funds decisions that need durable memory review

## Stop Condition
Disable or retire if it produces no useful update recommendations for 14 consecutive days or duplicates another documentation/memory cronjob.

## Approval Needed
No further approval needed if activated exactly as report-only. Reuben provided the cronjob spec directly. Any future auto-edit behavior requires separate explicit approval.
